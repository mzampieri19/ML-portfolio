## Project Overview

This project implements a **Deep Q-Network (DQN) agent** capable of learning to play games through reinforcement learning. The agent learns optimal strategies by interacting with environments and receiving rewards for good actions while being penalized for poor ones.

### Supported Environments

- **CartPole-v1**: Balance a pole on a cart by moving left or right
- **FlappyBird-v0**: Navigate a bird through pipes by controlling vertical movement
- **More Games**: Additional environments can be easily added

The implementation includes several advanced DQN techniques:

- **Double DQN**: Reduces overestimation bias in Q-value learning
- **Dueling DQN**: Separates state value from action advantages for better learning
- **Experience Replay**: Stores and reuses past experiences for more stable training
- **Target Networks**: Uses separate networks to stabilize training

### Key Achievements

- **Autonomous Gameplay**: Agent successfully plays both CartPole and Flappy Bird without human input
- **From Scratch Implementation**: Custom DQN implementation without using high-level RL libraries
- **Advanced Techniques**: Implements Double DQN and Dueling DQN architectures
- **Performance Tracking**: Real-time training progress and comprehensive logging

## Key Concepts

### Reinforcement Learning

The agent learns through trial and error by:

1. **Observing** the current state of the environment
2. **Taking actions** based on its current policy  
3. **Receiving rewards** for good actions and penalties for bad ones
4. **Learning** to maximize future rewards over time

### Q-Learning

Q-Learning estimates the "quality" (Q-value) of taking each possible action in any given state. The agent learns which actions lead to the highest cumulative rewards.

### Deep Neural Networks

Instead of storing Q-values in a table (impossible for complex states), a neural network approximates Q-values for any state-action pair.

### Exploration vs Exploitation

- **Exploration**: Taking random actions to discover new strategies (high epsilon)
- **Exploitation**: Using learned knowledge to take the best known actions (low epsilon)  
- The agent starts by exploring randomly and gradually shifts to exploiting learned knowledge

## Methodology

### 1. Experience Collection

The agent interacts with the environment using an epsilon-greedy strategy:

- With probability ε (epsilon), take a random action (exploration)
- With probability 1-ε, take the action with highest predicted Q-value (exploitation)
- Epsilon starts high (1.0 = 100% random) and gradually decreases during training

### 2. Experience Storage

All experiences (state, action, reward, next_state, done) are stored in a replay memory buffer. This allows the agent to:

- Learn from past experiences multiple times
- Break correlations between consecutive experiences
- Use data more efficiently

### 3. Network Training

Periodically, the agent samples random batches from replay memory and trains the neural network to:

- Predict Q-values for current states
- Minimize the difference between predicted and target Q-values
- Update weights using backpropagation

### 4. Target Network Updates

A separate "target network" provides stable Q-value targets during training. It's updated less frequently than the main network to prevent unstable learning.

## Model Architecture

### Standard DQN Architecture

```
Input Layer (State) → Hidden Layer (128-512 nodes) → Output Layer (Q-values for each action)
```

### Dueling DQN Architecture

The dueling architecture separates the Q-value estimation into two components:

```
Input Layer (State)
        ↓
Shared Hidden Layer (128-512 nodes)
        ↓
    ┌─────────┐    ┌──────────────┐
    │ Value   │    │ Advantage    │
    │ Stream  │    │ Stream       │
    └─────────┘    └──────────────┘
        ↓                ↓
    V(s)           A(s,a) for each action
        ↓                ↓
        └────────────────┘
                 ↓
        Q(s,a) = V(s) + A(s,a) - mean(A(s,·))
```

**Benefits of Dueling Architecture:**

- **Value Stream V(s)**: Estimates how good it is to be in a particular state
- **Advantage Stream A(s,a)**: Estimates how much better each action is compared to the average
- This separation helps the agent learn faster, especially when most actions have similar values

## DQN Implementation

### Neural Network Architecture

The DQN uses a simple feedforward neural network that takes the game state as input and outputs Q-values for each possible action.

<CodeBlock language="python" filename="dqn.py">
{`class DQN(nn.Module):
    def __init__(self, state_dim, action_dim, hidden_dim=256, enable_dueling_dqn=True):
        super(DQN, self).__init__()

        # Flag to enable dueling DQN architecture
        self.enable_dueling_dqn = enable_dueling_dqn

        # First fully connected layer: input state to hidden representation
        self.fc1 = nn.Linear(state_dim, hidden_dim)

        if self.enable_dueling_dqn:
            # Dueling DQN: separate streams for value and advantage

            # Value stream: hidden layer and output for state-value V(s)
            self.fc_value = nn.Linear(hidden_dim, 256)
            self.value = nn.Linear(256, 1)

            # Advantage stream: hidden layer and output for advantages A(s, a)
            self.fc_advantages = nn.Linear(hidden_dim, 256)
            self.advantages = nn.Linear(256, action_dim)

        else:
            # Standard DQN: single output layer for Q-values
            self.output = nn.Linear(hidden_dim, action_dim)

    def forward(self, x):
        # Pass input through first hidden layer with ReLU activation
        x = F.relu(self.fc1(x))

        if self.enable_dueling_dqn:
            # Dueling DQN forward pass

            # Value stream
            v = F.relu(self.fc_value(x))
            V = self.value(v)  # Shape: (batch_size, 1)

            # Advantage stream
            a = F.relu(self.fc_advantages(x))
            A = self.advantages(a)  # Shape: (batch_size, action_dim)

            # Combine value and advantage streams to get Q-values
            # Q(s, a) = V(s) + (A(s, a) - mean_a A(s, a))
            Q = V + A - torch.mean(A, dim=1, keepdim=True)

        else:
            # Standard DQN forward pass: directly output Q-values
            Q = self.output(x)

        return Q`}
</CodeBlock>

## Training Process

### Training Loop

The training process involves the agent playing many episodes of Flappy Bird, learning from its mistakes and gradually improving its strategy.

<CodeBlock language="python" filename="agent.py">
{`def optimize(self, mini_batch, policy_dqn, target_dqn):
        states, actions, new_states, rewards, terminations = zip(*mini_batch)

        # Stack tensors for batch processing
        states = torch.stack(states)
        actions = torch.stack(actions)
        new_states = torch.stack(new_states)
        rewards = torch.stack(rewards)
        terminations = torch.tensor(terminations).float().to(device)

        with torch.no_grad():
            # Compute target Q-values using Double DQN or standard DQN
            if self.enable_double_dqn:
                best_actions_from_policy = policy_dqn(new_states).argmax(dim=1)
                target_q = rewards + (1-terminations) * self.discount_factor_g * \
                                target_dqn(new_states).gather(dim=1, index=best_actions_from_policy.unsqueeze(dim=1)).squeeze()
            else:
                target_q = rewards + (1-terminations) * self.discount_factor_g * target_dqn(new_states).max(dim=1)[0]

        # Compute current Q-values for taken actions
        current_q = policy_dqn(states).gather(dim=1, index=actions.unsqueeze(dim=1)).squeeze()
        loss = self.loss_fn(current_q, target_q)

        # Backpropagation and optimizer step
        self.optimizer.zero_grad()
        loss.backward()
        self.optimizer.step()`}
</CodeBlock>

## Results and Performance

### Training Progress

The agent showed remarkable improvement over the training period:

- **Episodes 0-500**: Random exploration, average score < 5
- **Episodes 500-1000**: Learning basic game mechanics, average score 10-20
- **Episodes 1000-1500**: Consistent improvement, average score 50-80
- **Episodes 1500+**: Mastery achieved, average score > 100

### Key Metrics

- **Final Average Score**: 120+
- **Best Single Game**: 500+ 
- **Training Episodes**: 1,800 
- **Success Rate**: 95% (games with score > 50)

### Learning Insights

1. **Exploration vs Exploitation**: The epsilon-greedy strategy was crucial for balancing learning and performance
2. **Experience Replay**: Storing and replaying past experiences significantly improved learning stability
3. **Target Network**: Using a separate target network prevented training instability
4. **Reward Shaping**: Careful reward design encouraged the agent to focus on survival and scoring

## Technical Challenges

### State Representation

Finding the right state representation was crucial:
- **Initial Attempt**: Raw pixel input (too complex)
- **Final Solution**: Simplified 4-dimensional state vector
- **Key Features**: Bird position, velocity, distance to next pipe, height difference
